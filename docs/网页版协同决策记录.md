更新时间：2026-06-23

## 项目定位

本仓库用于开发个人本地看盘与量化研究软件，当前优先服务指数基金、行业 ETF、行业主题基金研究。

第一阶段目标：

```text
数据获取 -> 本地存储 -> 指标计算 -> 策略管理 -> 轻量回测 -> 报告展示 -> 桌面 UI
```

当前不接入真实自动下单，仅用于研究、学习和模拟。

## 当前限制

1. PySide6 桌面 UI 尚未开发。
2. AKShare 真实数据访问当前可能受网络代理影响。
3. 回测引擎当前仅支持单标的、日线、无杠杆。
4. 用户策略代码尚未做沙箱隔离。
5. 尚未实现 HTML/PDF 回测报告导出。
6. 尚未接入提醒系统、模拟账户和策略组合管理。

## 变更记录：2026-06-23 yfinance 限流修复

问题：

```text
看盘中心获取 SPY 行情时报错：yfinance returned no data for SPY
本地复现显示 yfinance 触发 YFRateLimitError: Too Many Requests
```

修复：

```text
1. YFinanceProvider 保留原 yfinance 下载路径。
2. 当 yfinance 返回空数据或异常时，自动 fallback 到 Yahoo chart JSON 接口。
3. fallback 返回 source=yahoo_chart，仍按统一行情 schema 保存到 DuckDB。
4. 新增 Yahoo chart payload 解析单元测试。
5. .gitignore 忽略本地 DuckDB 缓存，避免提交行情数据库。
```

验证：

```text

uv run pytest：17 passed
uv run python -m compileall src tests：通过
```

影响范围：

```text
仅影响 yfinance provider 的空数据兜底逻辑。
看盘中心、数据存储和旧接口保持兼容。
```
