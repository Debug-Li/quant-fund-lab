# 量观 Quant Lab 当前功能与开发工作说明

更新时间：2026-06-24

## 1. 项目定位

量观 Quant Lab 是一个本地运行的指数与行业基金量化研究平台，当前主线架构为：

```text
FastAPI 后端 + React/Vite/TypeScript 前端 + ECharts 图表 + 本地数据文件/DuckDB
```

平台面向学习、研究、看盘、回测、组合分析和信号模拟，不接入真实交易下单，不自动买卖。

## 3. 当前页面功能

### 3.1 总览

功能：

```text
1. 展示组合市值、日收益、累计收益、VaR、信号数量等指标。
2. 展示组合净值与基准走势。
3. 展示 ETF 市场热力图、风险指标、资产配置、信号提醒和运行日志。
4. 优先读取本地真实 ETF 数据；缺数据时自动回退 demo。
```

主要接口：

```text
GET /api/dashboard/overview
```

### 3.2 市场看盘

功能：

```text
1. 默认标的为 510300 沪深300ETF。
2. 展示 K 线、成交量、MA5/MA20/MA60、MACD、RSI。
3. 展示自选 ETF 池、板块/行业基金热力、模拟盘口、市场摘要和信号。
4. 顶部工具栏支持点击美股/A股/ETF，切换默认市场与默认标的。
5. 搜索框输入代码回车后刷新当前标的。
6. 刷新数据按钮会按当前上下文重新请求。
```

主要接口：

```text
GET /api/market/snapshot?symbol=510300&market=a股etf&period=1d
GET /api/market/watchlist
GET /api/market/sector-flow
GET /api/market/news
```

当前边界：

```text
1. 新闻源暂未接入真实资讯服务。
2. 盘口为研究用模拟盘口，不代表交易所实时盘口。
```

### 3.3 策略研究

功能：

```text
1. 读取本地 strategies/ 目录中的策略文件。
2. 展示策略列表、代码预览、基础参数和运行说明。
3. 支持策略 validate 接口检查策略是否存在。
```

主要接口：

```text
GET /api/strategy/list
GET /api/strategy/{strategy_id}
POST /api/strategy/{strategy_id}/validate
```

### 3.4 回测分析

功能：

```text
1. 接入本地 BacktestEngine。
2. 当前默认策略为 MA Cross。
3. 支持按顶部选中的 symbol、market、period 请求回测。
4. 展示总收益、年化收益、最大回撤、夏普、胜率、交易次数。
5. 展示净值曲线、参数寻优热力图、月度收益、收益分布和交易订单。
```

主要接口：

```text
POST /api/backtest/run
POST /api/backtest/optimize
GET /api/backtest/history
GET /api/backtest/result/{backtest_id}
```

当前边界：

```text
1. 已接入单标的 MA Cross 回测。
2. ETF 轮动回测的完整可视化仍可继续增强。
```

### 3.5 组合管理

功能：

```text
1. 基于 data/processed/etf_close.parquet 计算组合净值。
2. 基于动量信号生成研究用目标持仓。
3. 展示持仓、资产配置、因子暴露、相关性矩阵、风险贡献和调仓建议。
```

主要接口：

```text
GET /api/portfolio/overview
GET /api/portfolio/holdings
GET /api/portfolio/risk
GET /api/portfolio/rebalance/suggestion
```

当前边界：

```text
组合数据是研究模型组合，不是券商真实账户持仓。
```

### 3.6 信号中心

功能：

```text
1. 接入 quant_fund_lab.simulate.daily_signal.latest_signal。
2. 使用动量 + 趋势过滤生成 ETF 研究信号。
3. 展示买入信号、观察、风控提醒、目标权重、动量和收盘价。
```

主要接口：

```text
GET /api/signal/list
POST /api/signal/generate
```

### 3.7 数据中心

功能：

```text
1. 展示 AKShare、YFinance、DuckDB 数据源状态。
2. 展示本地数据集概览：etf_close.parquet、etf_daily.parquet、market_data.duckdb。
3. 展示质量检查：缺失值、重复日期、价格为 0、异常涨跌幅。
4. 前端可点击“下载真实 AKShare 数据”。
5. 前端可点击“生成本地演示数据”。
```

主要接口：

```text
GET /api/data/status
POST /api/data/download
POST /api/data/demo
GET /api/data/preview
POST /api/data/quality-check
```

当前边界：

```text
AKShare 依赖公开网页接口，可能受网络、代理和上游接口波动影响。
失败时页面保留 demo fallback，不会空白。
```

### 3.8 风险监控

功能：

```text
1. 基于 ETF close matrix 计算年化波动率。
2. 计算 VaR 95%、CVaR 95%、最大回撤、相关性均值和标的数量。
3. 展示回撤曲线、收益分布、相关性矩阵、风险贡献和风险提醒。
```

主要接口：

```text
GET /api/risk/overview
GET /api/risk/alerts
POST /api/risk/check
```

### 3.9 报告中心

功能：

```text
1. 扫描 reports/ 目录。
2. 展示 csv、parquet、html、md、图片等报告文件。
3. 支持 CSV/Parquet 预览。
```

主要接口：

```text
GET /api/report/list
GET /api/report/{report_id}
```

### 3.10 设置

功能：

```text
1. 展示项目路径、数据目录、报告目录、默认市场、默认标的。
2. 展示依赖诊断：fastapi、pandas、duckdb、akshare、yfinance、bt、streamlit。
```

主要接口：

```text
GET /api/settings
GET /api/settings/diagnostics
```

## 4. 顶部工具栏交互

当前顶部工具栏已从静态标签改为真实可点击控件。

功能：

```text
1. 美股：默认 SPY，market=US。
2. A股：默认 000001，market=a股股票。
3. ETF：默认 510300，market=a股etf。
4. 周期：支持日线、周线、月线。
5. 搜索框输入代码回车后刷新市场页。
6. 刷新数据按钮重新请求当前页面数据。
7. 运行回测按钮跳转回测页，并使用当前标的/市场/周期。
```

## 5. 数据与真实接口状态

当前真实数据优先级：

```text
1. 本地 DuckDB：data/market_data.duckdb。
2. 本地原始 ETF 数据：data/raw/etf_daily.parquet。
3. 本地 ETF close matrix：data/processed/etf_close.parquet。
4. AKShare 下载 ETF 研究池。
5. YFinance/Yahoo Chart 用于美股/ETF。
6. 如果真实数据不可用，则自动 fallback 到 demo data service。
```
